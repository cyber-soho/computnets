💡 Pro Tip: Shortcuts

  • Tab: Autocompletes commands (Type `sh ver` + Tab).
  • Ctrl+Z: Emergency exit! Jumps straight back to Privileged EXEC (`Router#`) from anywhere.
  • ?: Context-sensitive help. Shows you what you can type next.

Lesson 2: The "Russian Doll" Model

IOS uses a hierarchical structure. You must dive deeper into specific modes to change specific components, and exit back out to save or view status.

Visualizing the Hierarchy

User EXEC Mode (Router>)

Lobby area. Connect here first. Type enable to enter the next level.

Privileged EXEC Mode (Router#)

Manager view. Full visibility. Type conf t to configure.

Global Config (Router(config)#)

Change hostname, security. Branch out to specific sub-modes:

Interface Config
config-if
IPs, Ports
Line Config
config-line
Console, SSH

Entering Modes

  • To Privileged: enable
  • To Global Config: configure terminal
  • To Interface: interface Gi0/0
  • To Console Line: line console 0

Exiting Modes

  • Go Up One Level: exit
  • Jump to Router#: end or Ctrl+Z
  • Tip: Don't type 'exit' 3 times. Just use 'end'.

Lesson 3: The Memory Trinity

Cisco routers have three brains. Understanding which one is active is the difference between a successful config and a disastrous reboot.

RAM

Volatile
running-config

The whiteboard. Holds changes happening right now. Wipe the board (reboot), and it's all gone.

NVRAM

Non-Volatile
startup-config

The notebook. Permanent storage for configurations. Loaded into RAM at boot.

Flash

Non-Volatile
.bin (IOS Image)

The hard drive. Stores the operating system files itself.

Lesson 4: Device Hardening

A default router is an open door. Apply the layers of security to protect the device management plane. Never use `enable password` (plaintext); always use `enable secret` (encrypted).
