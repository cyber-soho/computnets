Lesson 1: Routing Tables

⏱ 15 min

What Is a Routing Table?

Every router — and every computer with a network connection — maintains an internal reference list called a routing table. It is stored in RAM (Random Access Memory) and is consulted for every single packet the device processes.

It answers one question over and over: "If a packet needs to go to this destination network, which door (interface) should I push it through, and who is the next person (next hop) I should hand it to?"

Routing Table Fields

FieldMeaning
Destination NetworkTarget network address, e.g. 192.168.10.0/24
Subnet Mask / PrefixDefines the size of the destination network
Next HopIP (Internet Protocol) address of the next router in the path
Exit InterfaceThe physical or logical port the packet leaves through (e.g., eth0, Gi0/1)
Metric / CostA numeric value representing "how expensive" this route is — lower is better
Route SourceHow the router learned the route: C (Connected), S (Static), O (OSPF — Open Shortest Path First), etc.

Longest Prefix Match — The Core Algorithm

When a packet arrives, the router:

Extracts the destination IP address from the packet header.

Scans the routing table and finds all networks that contain this destination.

Selects the most specific match — the entry with the longest prefix length.

Forwards the packet via that route's exit interface to the indicated next hop.

If two routes tie in prefix length, the one with the lower metric (cost) wins.

Real-Life Analogy

🏙️ The City Post Office

Task: Demonstrate how a routing table works by comparing it to the sorting process at a city post office.

Picture a post office. Letters arrive by the sack. A clerk reads the postal code on each envelope (like a router reading the destination IP), and sorts it into labelled bins (routing table entries). Each bin connects to a truck bay (exit interface) that goes to the next sorting depot (next hop). The "Everything else" bin is the default route. The clerk never opens the letter — they only read the outside.

💻 Code Example — Routing Table Simulator (Python)

Task: Build a Python script that simulates how a router's routing table works — receiving a destination IP and determining the correct exit interface using longest prefix match.

Python 3 
#!/usr/bin/env python3
# Routing Table Simulator — Lesson 1

# Import Python's built-in module for working with IP addresses and networks.
import ipaddress

# Define the routing table as a list of dictionaries.
# Each entry mirrors a real router's table row.
routing_table = [
    {"destination": "192.168.10.0/24", "next_hop": "10.0.0.1",
     "interface": "eth0", "metric": 10, "source": "C"},
    {"destination": "172.16.0.0/16",  "next_hop": "10.0.0.2",
     "interface": "eth1", "metric": 20, "source": "S"},
    {"destination": "10.10.10.0/24",  "next_hop": "10.0.0.3",
     "interface": "eth2", "metric": 15, "source": "O"},
    # Default route — "everything else" bin
    {"destination": "0.0.0.0/0",      "next_hop": "10.0.0.254",
     "interface": "eth3", "metric": 100, "source": "S"},
]

def lookup(destination_ip, table):
    # Convert the input string into an IPv4Address object.
    dest = ipaddress.IPv4Address(destination_ip)
    best_match = None
    longest_prefix = -1

    # Check every route entry for a match.
    for route in table:
        network = ipaddress.IPv4Network(route["destination"], strict=False)
        # Does the destination IP fall within this network?
        if dest in network:
            # Keep the most specific (longest prefix) match.
            if network.prefixlen > longest_prefix:
                best_match = route
                longest_prefix = network.prefixlen
    return best_match

# Test with several destination IPs.
for ip in ["192.168.10.45", "172.16.5.100", "8.8.8.8"]:
    result = lookup(ip, routing_table)
    print(f"Packet to {ip} → {result['interface']} via {result['next_hop']}")

Import: The ipaddress module parses CIDR (Classless Inter-Domain Routing) notation and tests if an IP falls in a range.

Table definition: Four route entries as dictionaries — destination, next hop, interface, metric, and source code.

Lookup function: Converts the destination IP to an object, loops through routes, checks containment, and tracks the longest prefix match.

Test loop: Three IPs are tested. The first two match specific routes; 8.8.8.8 (Google DNS) falls through to the default route on eth3.

Every router consults its routing table for every packet. The longest prefix match algorithm ensures the most specific route always wins. Without a routing table, a router cannot forward anything.

Lesson 2: Static Routing Logic

⏱ 15 min

What Is Static Routing?

Static routing is the simplest form of routing. A network administrator manually types each route into the router's configuration. The router does not learn anything on its own.

On a Cisco IOS (Internetwork Operating System) router:

Cisco IOS
ip route 192.168.20.0 255.255.255.0 10.0.0.2

Translation: "Send any packet destined for 192.168.20.0/24 to the router at 10.0.0.2."

✅ Advantages

  • ● Zero overhead — no protocol traffic generated
  • ● Predictable — admin knows exactly where packets go
  • ● Simple — easy for small networks
  • ● Secure — no route advertisements to intercept

❌ Disadvantages

  • ● No fault tolerance — cannot adapt to link failures
  • ● Unscalable — hundreds of entries for large networks
  • ● Error-prone — one typo creates a black hole
  • ● Maintenance burden — every change is manual
Real-Life Analogy

import ipaddress

# Define three static routes for a branch office.
routes = [
    {"dest": "192.168.20.0", "mask": "255.255.255.0",
     "hop": "10.0.0.2", "desc": "Branch Office LAN"},
    {"dest": "172.16.50.0",  "mask": "255.255.255.0",
     "hop": "10.0.0.3", "desc": "Data Centre"},
    {"dest": "0.0.0.0",      "mask": "0.0.0.0",
     "hop": "10.0.0.1", "desc": "Default to ISP"},
]

# Generate Cisco IOS commands.
for r in routes:
    print(f"ip route {r['dest']} {r['mask']} {r['hop']}")

# Simulate link failure: 10.0.0.2 goes down.
print("\n⚠️  LINK FAILURE: 10.0.0.2 is DOWN")
for r in routes:
    if r["hop"] == "10.0.0.2":
        print(f"  ❌ AFFECTED: {r['desc']} — packets DROPPED")
    else:
        print(f"  ✅ OK: {r['desc']}")

Static routing is simple, predictable, and has zero overhead — but it cannot heal itself. Best for small, stable networks or stub connections. For anything larger, dynamic routing is essential.

Lesson 3: The Function of a Default Gateway

⏱ 15 min

What Is a Default Gateway?

Every device on a network has a configuration parameter called the default gateway. It is the IP (Internet Protocol) address of the router that the device should send packets to when the destination is outside its own local network.

Without a default gateway, a device is an island: it can talk to neighbors on the same subnet, but it cannot reach the broader internet.

Local vs. Remote Communication

ScenarioMechanismGateway Needed?
Same subnet (laptop → printer at .50)Layer 2 — MAC (Media Access Control) addressNo
Different network (laptop → google.com)Layer 3 — Routed via gatewayYes

Gateway Configuration by OS (Operating System)

OSMethod
Windowsipconfig / DHCP (Dynamic Host Configuration Protocol)
Linuxip route show default / DHCP
macOSroute -n get default / DHCP
Cisco Switchip default-gateway <address>
Mobile DevicesDHCP from Wi-Fi router (automatic)

💻 Code Example — Default Gateway Detector & Tester (Python)

Task: Detect the system's default gateway and test reachability — the first troubleshooting step when internet access fails.

Python 3
import subprocess, platform, re, sys

def get_default_gateway():
    # Detect the OS and run the appropriate command.
    os_name = platform.system()
    if os_name == "Linux":
        result = subprocess.run(["ip","route","show","default"],
                                 capture_output=True, text=True)
        match = re.search(r"default via (\S+)", result.stdout)
    elif os_name == "Windows":
        result = subprocess.run(["ipconfig"], capture_output=True, text=True)
        match = re.search(r"Default Gateway.*?:\s*(\d+\.\d+\.\d+\.\d+)", result.stdout)
    return match.group(1) if match else None

# Detect and test the gateway.
gw = get_default_gateway()
if gw:
    print(f"✅ Gateway found: {gw}")
    # Ping the gateway to check reachability.
    flag = "-n" if platform.system() == "Windows" else "-c"
    ok = subprocess.run(["ping", flag, "3", gw],
                         capture_output=True).returncode == 0
    print("✅ Reachable!" if ok else "❌ Unreachable!")
else:
    print("❌ No gateway configured — local only!")

📐 Troubleshooting Flowchart

Is a default gateway configured? → NO: Check DHCP / manual IP settings

Can you PING the gateway? → NO: Check cable, Wi-Fi, router power

Can you PING an external IP like 8.8.8.8? → NO: Upstream / ISP (Internet Service Provider) problem

Can you PING a domain like www.google.com? → NO: DNS (Domain Name System) issue

All YES → Everything works! 🎉

The default gateway is the "reception desk" for every device on a network. First troubleshooting step: can you reach it? No gateway = no internet. Unreachable gateway = check physical layer.

Lesson 4: Dynamic Routing Explained

⏱ 15 min

What Is Dynamic Routing?

Dynamic routing is the opposite of static routing. Instead of a human typing every route, routers talk to each other using a routing protocol — a set of rules that let them share which networks they can reach, how far away those networks are, and which paths are working.

When a link fails, routers detect it, exchange updates, and recalculate their tables — all without human intervention. This is the GPS navigation app from our earlier analogy.

How It Works — Four Steps

Neighbor Discovery: Routers send "hello" messages to find adjacent routers running the same protocol.

Information Exchange: Routers share their known routes (or link states) with their neighbors.

Route Computation: Each router independently calculates the best path to each destination.

Ongoing Updates: Routers send new updates periodically or upon detecting a change.

Key Dynamic Routing Protocols

ProtocolFull NameTypeKey IdeaBest For
RIP Routing Information ProtocolDistance Vector Counts hops (max 15); simple but slowSmall LANs
OSPF Open Shortest Path FirstLink State Full map + Dijkstra's algorithm; fastEnterprise
EIGRP Enhanced Interior Gateway Routing ProtocolHybrid DUAL (Diffusing Update Algorithm); Cisco-developedCisco networks
BGP Border Gateway ProtocolPath Vector Routes across the entire internetISPs / Internet

🚌 City Bus Network with Live Rerouting

Task: Explain dynamic routing by comparing it to a smart city bus system.

Four bus depots (routers) are connected by roads (links). They radio each other every few minutes. One day the bridge between Depot A and Depot C collapses. With static schedules, buses keep driving to the bridge and passengers are stranded. With live rerouting, Depot A immediately radios Depot B: "Bridge is down, relay passengers via the highway." Within seconds, a new route is in place.

💻 Code Example — Dijkstra's Algorithm Simulator (Python)

Task: Simulate how OSPF finds the shortest path and automatically recalculates when a link fails.

Python 3
import heapq, copy

class Network:
    def __init__(self):
        # Adjacency list: router → [(neighbor, cost), ...]
        self.graph = {}

    def add_link(self, a, b, cost):
        # Add bidirectional link (full-duplex).
        self.graph.setdefault(a, []).append((b, cost))
        self.graph.setdefault(b, []).append((a, cost))

    def remove_link(self, a, b):
        # Simulate link failure — remove edge in both directions.
        self.graph[a] = [(n,c) for n,c in self.graph[a] if n != b]
        self.graph[b] = [(n,c) for n,c in self.graph[b] if n != a]

    def dijkstra(self, source):
        # Dijkstra's shortest-path algorithm using a min-heap.
        dist = {r: float("inf") for r in self.graph}
        prev = {r: None for r in self.graph}
        dist[source] = 0
        pq = [(0, source)]
        while pq:
            cost, node = heapq.heappop(pq)
            if cost > dist[node]: continue
            for nbr, w in self.graph[node]:
                if cost + w < dist[nbr]:
                    dist[nbr] = cost + w
                    prev[nbr] = node
                    heapq.heappush(pq, (dist[nbr], nbr))
        return dist, prev

# Build a 4-router network, all links cost 1.
net = Network()
net.add_link("A","B",1); net.add_link("A","C",1)
net.add_link("B","D",1); net.add_link("C","D",1)

dist, _ = net.dijkstra("A")
print("BEFORE:", dist)  # A→C = 1

# Simulate link failure A↔C.
net_fail = copy.deepcopy(net)
net_fail.remove_link("A", "C")
dist2, _ = net_fail.dijkstra("A")
print("AFTER:", dist2)  # A→C = 3 (via B→D→C)

Dynamic routing protocols let routers self-heal without human intervention. They discover neighbors, share information, compute best paths, and adapt to failures automatically. The trade-off: bandwidth and CPU — but the resilience is worth it.
