Lesson 1: Routing Tables

This section explores the internal reference list stored in a router's RAM. Without this table, a router cannot forward packets. Click through the routing table fields below to understand the criteria routers use to answer the question: "Where do I send this packet next?"

🏢 The Post Office Analogy

  • Router = Post office sorting room
  • Destination IP = ZIP code on envelope
  • Routing table rows = Labelled sorting bins
  • Exit interfaces = Truck loading bays
  • Default route = "Everything else" bin

The Core Forwarding Algorithm

1. Extract destination IP from packet header
2. Compare against every route in the table
3. Find all matching networks
4. Select the most specific match (Longest Prefix)
5. Forward packet via that route's exit interface

# If prefix lengths tie → choose route with lower metric

Lesson 2: Static Routing Logic

Static routes are manually configured paths. In this section, compare the specific scenarios where static routing excels against its inherent limitations. Use the toggle to switch between analyzing the advantages and disadvantages of this zero-overhead, yet rigid, method.

Cisco IOS Command

ip route <network> <mask> <next_hop>

Example: ip route 192.168.20.0 255.255.255.0 10.0.0.2

Zero Overhead

No routing protocol traffic generated on the network.

Predictable

Admin knows exactly where packets go every time.

Simple

Easy to understand and configure for small networks.

Secure

No route advertisements broadcasted to intercept.

No Fault Tolerance

Cannot adapt or reroute if a link fails.

Unscalable

500 subnets means hundreds of manual entries per router.

Error-Prone

One typo creates a routing black hole.

Maintenance Burden

Every single topology change requires manual admin intervention.

Best for: Small offices (2-3 routers), Stub networks (one way in/out), or establishing a Default Route to an ISP.

Lesson 4: Dynamic Routing

Dynamic routing protocols allow networks to self-heal. This section breaks down the major protocols. Explore the interactive radar chart to understand the trade-offs between simplicity, scalability, and speed among different dynamic routing protocols based on their functional types.

Distance Vector (RIP)

"Passing rumors." Shares table with neighbors. Counts hops.

Best for: Small LANs

Link State (OSPF)

Builds full map via LSDB. Uses Dijkstra's algorithm.

Best for: Enterprise

Path Vector (BGP)

Shares full path across organizations.

Best for: Internet / ISPs

The 4-Step Dynamic Process

  1. Neighbor Discovery — Routers send "hello" messages to adjacent devices.
  2. Information Exchange — Share known routes or link states.
  3. Route Computation — Each router calculates best paths independently.
  4. Ongoing Updates — Periodic/event-triggered updates keep routing tables current.
