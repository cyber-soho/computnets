Architecture Overview

The Topology You Are Building

Layer 3 Switching moves inter-VLAN (Virtual Local Area Network) routing inside the switch hardware, eliminating the bandwidth bottleneck of Router-on-a-Stick (RoaS).

Core Content

Theory, Configuration & Verification

The one-line summary: An SVI (Switched Virtual Interface) is a virtual gateway you create on a multilayer switch — one per VLAN. When you enable ip routing, the switch routes between SVIs in hardware, without involving any external router.

RoaS vs. Layer 3 Switching

Criterion Router-on-a-Stick (RoaS) Layer 3 Switching (SVI)
Hardware neededRouter + Layer 2 switchOne multilayer switch
BandwidthBottleneck — all traffic through one uplinkWire-speed — routed in hardware ASIC
ScalabilityLow — one uplink = single point of congestionHigh — dedicated routing engine
Typical useSmall branch, lab environmentsCampus distribution/core layer
Config complexityLowModerate
CostLower initial hardware costHigher (L3 switch more expensive than L2)

How the ip routing Command Works

A multilayer switch ships in Layer 2 mode by default. The single command ip routing activates the internal routing engine. Without it, SVIs exist but cannot forward traffic between subnets — they behave as simple management interfaces only.

What Makes an SVI Come Up?

All three conditions must be true simultaneously:

CONDITION 1
VLAN exists in DB
Verify with show vlan brief
CONDITION 2
At least one active port
A port assigned to that VLAN must be up
CONDITION 3
SVI not shutdown
no shutdown must be issued

Audit Your Existing Topology
~20 MIN · BEFORE YOU CHANGE ANYTHING
✓

Before making any changes, document the current state of your network. These outputs become your "before" baseline for the report.

On the router:

Router# show running-config Router# show ip interface brief Router# show ip route

On each switch:

Switch# show vlan brief Switch# show interfaces trunk

Save or screenshot all outputs. Answer in your lab notes: which physical interface carries the trunk? What VLAN sub-interfaces exist on the router? Can you ping across VLANs right now?

All three VLANs visible in show vlan brief. Trunk links active. Baseline documented.
02
Confirm Your Switch Supports Layer 3
~15 MIN
✓

Not all switches can route. Verify the distribution switch in your topology is a multilayer device with IP Services IOS (Internetwork Operating System) feature set.

Switch# show version

Look for "IP Services" in the IOS feature set. In Cisco Packet Tracer, use a 3560 or 3650 switch model.

Research prompt: What is the difference between "IP Base" and "IP Services" IOS feature sets? What routing protocols does each support? Write two sentences in your notes.

Confirmed: switch supports IP routing. Model/IOS version noted.
03
Remove Router-on-a-Stick Configuration
~20 MIN · AVOID DUPLICATE GATEWAYS
✓

Remove sub-interfaces from the router and shut down the physical trunk link. Leaving both active creates duplicate gateways and unpredictable routing.

Router# configure terminal ! Remove each VLAN sub-interface Router(config)# no interface GigabitEthernet0/0.10 Router(config)# no interface GigabitEthernet0/0.20 Router(config)# no interface GigabitEthernet0/0.30 ! Shut down the trunk uplink Router(config)# interface GigabitEthernet0/0 Router(config-if)# shutdown Router(config-if)# end Router# write memory
show ip interface brief on the router shows trunk interface as down/down. No VLAN sub-interfaces exist.
04
Enable IP Routing on the Switch
~10 MIN · THE MOST IMPORTANT COMMAND
✓

This single command activates the routing engine on the multilayer switch. Without it, nothing in the next steps will work.

L3Switch# configure terminal L3Switch(config)# ip routing L3Switch(config)# end L3Switch# write memory ! Verify: L3Switch# show running-config | include ip routing
ip routing appears in the running configuration output.
05
Create SVIs for Each VLAN
~25 MIN · USE YOUR PHASE 2 IP ADDRESSES
✓

Create one Switched Virtual Interface (SVI) per VLAN. Use the IP addresses from your Phase 2 VLSM (Variable Length Subnet Mask) plan. The examples below use /24 for illustration — use your actual subnets.

L3Switch(config)# interface vlan 10 L3Switch(config-if)# description HR_Gateway L3Switch(config-if)# ip address 192.168.10.1 255.255.255.0 L3Switch(config-if)# no shutdown L3Switch(config-if)# interface vlan 20 L3Switch(config-if)# description Engineering_Gateway L3Switch(config-if)# ip address 192.168.20.1 255.255.255.0 L3Switch(config-if)# no shutdown L3Switch(config-if)# interface vlan 30 L3Switch(config-if)# description Guest_Gateway L3Switch(config-if)# ip address 192.168.30.1 255.255.255.0 L3Switch(config-if)# no shutdown L3Switch(config-if)# end L3Switch# write memory ! Verify all SVIs are up: L3Switch# show ip interface brief
All three SVIs show up/up in show ip interface brief.
06
Update Default Gateway on End Hosts
~15 MIN
✓

Every PC in your topology must now use the SVI address as its default gateway. In Packet Tracer: open each PC → Desktop → IP Configuration → update Default Gateway. If using DHCP (Dynamic Host Configuration Protocol) pools:

L3Switch(config)# ip dhcp pool VLAN10_POOL L3Switch(dhcp-config)# default-router 192.168.10.1
Each PC shows the correct SVI address as its default gateway.
07
Verify Inter-VLAN Connectivity
~30 MIN · THE PAYOFF
✓

Test bidirectional connectivity between every VLAN pair. From a PC in VLAN 10, ping hosts in VLAN 20 and VLAN 30.

! From a host in VLAN 10: ping 192.168.20.x → host in Engineering ping 192.168.30.x → host in Guest ! Check routing table on switch: L3Switch# show ip route ! Expected output (three connected routes): C 192.168.10.0/24 is directly connected, Vlan10 C 192.168.20.0/24 is directly connected, Vlan20 C 192.168.30.0/24 is directly connected, Vlan30 ! Traceroute (note: 1 hop to reach destination) traceroute 192.168.20.x
Pings succeed between all three VLANs. Routing table shows three directly connected routes. Traceroute reaches destination in one hop.
08
Capture & Analyse One Packet
~20 MIN · CONNECT THEORY TO REALITY
✓

Use Packet Tracer simulation mode or Wireshark / tcpdump in GNS3. Capture a ping from VLAN 10 to VLAN 20 and observe what changes at each layer.

What to observe and annotate:

  • The source PC ARPs (Address Resolution Protocol) for the gateway address (.1), not the destination host.
  • The switch replies with its SVI MAC (Media Access Control) address.
  • The ICMP (Internet Control Message Protocol) echo request arrives at the switch inside a frame addressed to the SVI's MAC.
  • The switch re-encapsulates the packet with VLAN 20's SVI MAC as the new source.
  • Key observation: the source/destination IP never changes — only the MAC addresses change at each Layer 2 hop.
Report requirement: Include at least one annotated packet capture screenshot showing Layer 2 (MAC) and Layer 3 (IP) headers from the same packet.
Captured and explained one complete inter-VLAN packet exchange. Screenshot saved for the report.

Debug Guide

When Things Don't Work

Always debug bottom-up: Layer 1 (cables/links) → Layer 2 (VLANs, trunks) → Layer 3 (routing, SVIs). Never blame Layer 3 before confirming Layer 2 is healthy.

Symptom
SVI stuck at down/down
Likely Cause
VLAN not in database, or no active ports assigned to it
Diagnostic
show vlan brief
Symptom
SVI is up/down
Likely Cause
No active physical port assigned to that VLAN
Diagnostic
show interfaces trunk
show vlan brief
Symptom
Ping to gateway fails
Likely Cause
Host default gateway set incorrectly, or SVI in shutdown state
Diagnostic
show ip interface brief
Symptom
Ping between VLANs fails
Likely Cause
ip routing not enabled on the switch
Diagnostic
show run | include ip routing
