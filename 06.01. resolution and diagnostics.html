Lesson 1

Address Resolution Protocol (ARP)

Mapping Layer 2 to Layer 3 (RFC 826)

The Problem ARP Solves

This section explains the critical bridge between IP and MAC addresses. You will interact with a simulation of the 4-step ARP process to understand how a device dynamically discovers the physical address of a peer on the same local network when it only knows its logical IP address.

  • ▶ Computer A wants to send data to B on the same LAN.
  • ▶ A knows B's IP address, but not B's MAC address.
  • ▶ Ethernet frames require a destination MAC.
Security Risk: ARP has no authentication. It is vulnerable to Spoofing/Poisoning, enabling Man-in-the-Middle (MitM) attacks.

Lesson 2

Internet Control Message Protocol (ICMP)

Ping & Traceroute (RFC 792)

This section covers the network's built-in diagnostic language. You will explore how ICMP enables tools like Ping to verify reachability and Traceroute to map network paths. The interactive chart below simulates a Ping test, visually demonstrating how Round-Trip Time (RTT) fluctuates, a key metric for diagnosing network latency.

Ping: "Are You There?"

Sends an Echo Request (Type 8), waits for an Echo Reply (Type 0).

  • RTT: Total delay for request + reply.
  • TTL: Remaining hop count.
  • Packet Loss: % of requests with no reply.

Traceroute: Path Discovery

Exploits the TTL (Time To Live) field. Sends packets with increasing TTLs to force routers along the path to reply with Time Exceeded (Type 11) messages.

Simulated Ping (RTT Variance)

Systematic Troubleshooting Framework

Rule: Always test from nearest to farthest to narrow the fault domain.

Step 1
ping <gateway>

Tests local LAN link.
Failure: Cable/Wi-Fi issue.

Step 2
ping 8.8.8.8

Tests Internet path.
Failure: ISP issue.

Step 3
ping google.com

Tests DNS resolution.
Failure: DNS server issue.

Step 4
traceroute 8.8.8.8

Maps full path.
Identifies exact failing hop.

Don't trust ICMP blindly: Firewalls often block it.
