The Pillars of Network Connectivity

This section introduces the foundational concepts of network communication. Before diving into protocols and bits, it is essential to understand the roles of IP, DHCP, and DNS. Together, these three services form a complete chain that allows your device to connect to any service on the global internet. The analogy below provides a conceptual map for the technical details in the following tabs.

The Complete Chain

DHCP ➔ DNS ➔ IP ➔ Connection

DHCP assigns your IP address and tells you which DNS server to use. DNS resolves every domain name you type into an IPv4 or IPv6 address. Your browser then uses that IP to establish the connection.

Lesson 1: IPv4 and IPv6 — The Addressing System

Every device requires a unique identifier on a network. Currently, two coexisting protocols provide this: the legacy IPv4, which is suffering from address exhaustion, and the modern IPv6, designed to provide limitless addresses and built-in security features. Explore the data and structural differences below.

🧮 IPv4 (Internet Protocol v4)

  • Standardised: 1981 (RFC 791)
  • Size: 32-bit address space
  • Notation: Dotted-decimal (e.g., 192.168.1.10)
  • Structure: Four octets (8 bits each), range 0–255. Variable header with checksum.
  • Status: Global pool exhausted in 2011. Relies on workarounds like NAT (Network Address Translation) and CIDR.

Key IPv4 Concepts

Subnet Mask: Divides address into network vs. host portions.

Default Gateway: Router address for exiting the local subnet.

Private Ranges: Non-routable IPs (e.g., 10.x.x.x, 192.168.x.x) for local networks.

🚀 IPv6 (Internet Protocol v6)

  • Standardised: RFC 8200
  • Size: 128-bit address space
  • Notation: Colon-hexadecimal (e.g., 2001:0db8:85a3::8a2e:0370:7334)
  • Structure: Eight groups of four hex digits. Fixed 40-byte header, no checksum for faster routing.
  • Features: No NAT required. Integral IPsec security. Built-in auto-configuration (SLAAC).

Simplification Rules

1. Drop leading zeroes: 0db8 becomes db8

2. Collapse ONE all-zero sequence: 0000:0000:0000 becomes ::

The Exhaustion Problem: Address Space Comparison

A logarithmic scale is necessary to visualise the difference. IPv4 provides ~4.3 Billion. IPv6 provides ~3.4 × 10³⁸.

Key Takeaway: We exist in a dual-stack network era. IPv4 is running out, making NAT a temporary band-aid. IPv6 offers a permanent, limitless, and secure supply.

Lesson 2: DHCP — Automatic Configuration

Manually configuring 15,000 devices on a campus network is impossible. The Dynamic Host Configuration Protocol (DHCP) solves this by automating IP address assignment. In IPv4, this is achieved through the rapid, four-step DORA handshake. In IPv6, networks can choose between DHCPv6 or serverless autoconfiguration (SLAAC).

Discover Phase

The device has no IP address yet. It broadcasts to 255.255.255.255 asking: "Is there a DHCP server out there?"

📚 DHCP Dictionary

Lease Time The duration the IP is rented. The client must renew it or lose connectivity. Addresses are leased to enable efficient reuse.
Scope / Pool The range of assignable addresses configured on the server (e.g., .50 through .200).
Reservation A permanent mapping in the server tying a specific MAC address to a specific IP.
Relay Agent A network device that forwards DHCP broadcast messages across different subnets.

🌐 IPv6 Auto-Config

IPv6 introduces two distinct approaches for address assignment, reducing reliance on central servers.

SLAAC Stateless Address Autoconfiguration

Routers advertise the network prefix via RA (Router Advertisement) messages. Hosts independently generate their own interface ID. No central DHCP server is required.

DHCPv6 RFC 8415

Functions similarly to IPv4 DHCP. Used when network administrators require tighter control, specific DNS assignments, or address tracking.

Lesson 3: DNS Operations — Name Resolution

Computers communicate using IP addresses, but humans use domain names (like www.google.com). The Domain Name System (DNS) acts as the internet's phonebook, translating these human-friendly names into routable IPs through a globally distributed, hierarchical system.

The DNS Hierarchy

DNS is an inverted tree structure. Queries cascade downwards to find the specific authoritative answer.

DNS Security

  • Spoofing/Poisoning: Attacker injects false IP records into caches.
  • DNSSEC: Adds cryptographic signatures to records to prevent spoofing.
  • DoH & DoT: Encrypts DNS queries over HTTPS or TLS for privacy.
