Transport Layer & Access Control

Before configuring routers, you must understand the nature of the traffic you are filtering. Access Control Lists (ACLs) rely heavily on differentiating between Layer 4 protocols and understanding wildcard masks.

Reliable & Ordered

TCP is like a registered letter. It guarantees delivery through sequence numbers and acknowledgements. If a packet is lost, it is retransmitted. This requires overhead.

Common Ports

80 : HTTP
443 : HTTPS
22 : SSH
21 : FTP
25 : SMTP

The 3-Way Handshake

CLIENTSERVER
SYN → "Hello"
"Ready" ← SYN-ACK
ACK → "Let's go"

Fast & Stateless

UDP is like a postcard. It sends data without establishing a connection or guaranteeing delivery. It is preferred for real-time applications where a missing packet is better than a delayed one.

Common Ports

53 : DNS
67/68: DHCP
69 : TFTP
123 : NTP
5004: RTP

Stateless Transmission

CLIENTSERVER
DATA → No handshake
DATA → Just stream
Packet Lost (Ignored)

Standard ACLs (1-99)

Filter based only on Source IP Address.

access-list 10 deny 192.168.30.0 0.0.0.255
access-list 10 permit any
Placement: Close to Destination

Extended ACLs (100-199)

Filter based on Source, Dest, Protocol, and Port.

permit tcp 192.168.10.0 0.0.0.255 host 192.168.50.10 eq 80
deny tcp 192.168.10.0 0.0.0.255 host 192.168.50.10 eq 22
Placement: Close to Source
█
The Implicit Deny: Every ACL ends with an invisible deny any any rule. If a packet matches no explicit rules, it is dropped.

Standard ACL Implementation

Blocks Guest VLAN (192.168.30.0) from reaching the server subnet entirely. Applied OUTBOUND on server interface.

! Enter global configuration mode Router(config)# access-list 10 deny 192.168.30.0 0.0.0.255 Router(config)# access-list 10 permit any ! Apply outbound on server-facing interface Router(config)# interface GigabitEthernet0/1 Router(config-if)# ip access-group 10 out

Extended ACL Implementation

Permits HTTP but denies SSH from Engineering VLAN to Server. Applied INBOUND on Engineering interface.

Router(config)# ip access-list extended ENGINEERING_POLICY Router(config-ext-nacl)# permit tcp 192.168.10.0 0.0.0.255 host 192.168.50.10 eq 80 Router(config-ext-nacl)# deny tcp 192.168.10.0 0.0.0.255 host 192.168.50.10 eq 22 Router(config-ext-nacl)# permit ip any any Router(config)# interface GigabitEthernet0/0.10 Router(config-if)# ip access-group ENGINEERING_POLICY in
Router# show access-lists Standard IP access list 10 10 deny 192.168.30.0, wildcard bits 0.0.0.255 (4 matches) 20 permit any (12 matches) Extended IP access list ENGINEERING_POLICY 10 permit tcp 192.168.10.0 0.0.0.255 host 192.168.50.10 eq www (8 matches) 20 deny tcp 192.168.10.0 0.0.0.255 host 192.168.50.10 eq 22 (2 matches) 30 permit ip any any (15 matches)

Note: Check the "matches" counter (hit count). If it reads zero after testing, your ACL is either applied to the wrong interface or evaluating logic incorrectly.
