OSI Layer 4

The Engine of the Internet

Welcome to the Transport Layer. Sitting right between the Network Layer (where packets go) and the Application Layer (what uses them), this is the layer that decides how your data gets there. It handles whether data arrived safely, in what order, and directs traffic to the correct application on your machine. Without Layer 4, there is no streaming, no email, no banking, and no gaming.

TCP vs. UDP: The Two Philosophies

This section contrasts the two dominant transport protocols. You will explore their core differences, overhead costs, and specific use cases. Understanding when to choose reliable delivery over raw speed is fundamental to network engineering.

📦

TCP (Transmission Control Protocol)

"The Certified Letter"

  • ✓ Connection-oriented: Requires a 3-way handshake (SYN → SYN-ACK → ACK).
  • ✓ Reliable & Ordered: Every byte is tracked, acknowledged, and reassembled in sequence.
  • ✓ Slower: Trades raw speed for absolute guarantees.
  • ✓ Use Cases: Web (HTTP/HTTPS), Email (SMTP/IMAP), File Transfer, Banking.
📩

UDP (User Datagram Protocol)

"The Postcard"

  • ⚡ Connectionless: No setup, just starts blasting datagrams.
  • ⚡ Best-effort: No retries, no acknowledgements, unordered delivery.
  • ⚡ Faster: Extremely lightweight with minimal overhead.
  • ⚡ Use Cases: VoIP, Online Gaming, Live Video, DNS queries.

Header Overhead Comparison

Visualizing the structural weight of each protocol segment/datagram.

Key Insight: TCP's header (minimum 20 bytes) carries sequence numbers, acknowledgment numbers, flags, and window sizes to ensure reliability. UDP's header (8 bytes) only carries source/destination ports, length, and a checksum. Minimalism enables low latency.

Module 8 Quick Review

📝 6 Facts to Memorize

  • Throughput Formula: Window Size ÷ RTT
  • TCP Header: Minimum 20 bytes
  • UDP Header: Exactly 8 bytes
  • Port Range: 0 to 65,535 (16-bit)
  • TCP Handshake: SYN → SYN-ACK → ACK
  • TCP Teardown: FIN → ACK → FIN → ACK

💡 Core Concepts

Flow vs. Congestion Control

Flow control protects the receiver (via the receive window). Congestion control protects the network.

Ephemeral Ports

Assigned automatically by the OS for outbound client connections (49152 - 65535). No registration needed.

Modern Protocols

QUIC (HTTP/3) runs on UDP for speed, but builds TCP-like reliability into the application space.
